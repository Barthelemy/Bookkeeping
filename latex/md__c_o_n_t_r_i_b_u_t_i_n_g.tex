A {\bfseries{resource can be a singleton or a collection}}. For example, \char`\"{}\`{}customers\`{}\char`\"{} is a collection resource and \char`\"{}\`{}customer\`{}\char`\"{} is a singleton resource (in a banking domain). We can identify \char`\"{}\`{}customers\`{}\char`\"{} collection resource using the URI \char`\"{}\`{}customers\`{}\char`\"{}. We can identify a single \char`\"{}\`{}customer\`{}\char`\"{} resource using the URI \char`\"{}\`{}/customers/\{customer\+Id\}\`{}\char`\"{}.

A {\bfseries{resource may contain sub-\/collection resources}} also. For example, sub-\/collection resource \char`\"{}\`{}accounts\`{}\char`\"{} of a particular \char`\"{}\`{}customer\`{}\char`\"{} can be identified using the URI \char`\"{}\`{}/customers/\{customer\+Id\}/accounts\`{}\char`\"{} (in a banking domain). Similarly, a singleton resource \char`\"{}\`{}account\`{}\char`\"{} inside the sub-\/collection resource \char`\"{}\`{}accounts\char`\"{}{\ttfamily can be identified as follows\+: \char`\"{}$<$/tt$>$/customers/\{customer\+Id\}/accounts/\{account\+Id\}\`{}\char`\"{}.}

{\ttfamily We have very precise rules over how our git commit messages can be formatted. This leads to {\bfseries{more readable messages}} that are easy to follow when looking through the {\bfseries{project history}}. But also, we use the git commit messages to {\bfseries{generate the change log}}.}
